JavaScript OO


  • JSON
    • {atributo1: 'valor', metodo1: function(){}}
  • Funciones
    • Con una función se define el constructor de un objeto. Se pueden agregar atributos en el metodo o usando Prototype.
  • new Object()
    • Con esta llamada se crea un objeto, al cual se le pueden agregar atributos y funciones.
Understanding JS OOP

JavaScript OO - JSON Ejemplo 

var rectanguloLit = { 
    lado1 : 10,
    lado2 : 20,
    area: function() {
        return this.lado1 * this.lado2;
    },
    perimetro: function() {
        return (this.lado1 * 2) + (this.lado2 * 2);
    }
};
http://jsfiddle.net/tPLhF/

JavaScript OO - Función 

var RectanguloFun = function() {
    this.lado1 = 10;
    this.lado2 = 20;
};

RectanguloFun.prototype.area = function() {
    return this.lado1 * this.lado2;
};

RectanguloFun.prototype.perimetro = function() {
    return (this.lado1 * 2) + (this.lado2 * 2);
};
http://jsfiddle.net/XDU6C/

JavaScript OO - new Object() 

var rectanguloObj = new Object();
rectanguloObj.lado1 = 10;
rectanguloObj.lado2 = 20;
rectanguloObj.area = function() {
    return this.lado1 * this.lado2;
};

rectanguloObj.perimetro = function() {
    return (this.lado1 * 2) + (this.lado2 * 2);
};
http://jsfiddle.net/HQr7e/

Backbone JS



  • Permite desarrollar aplicaciones single-page
  • Provee una estructuración de los archivos JS
  • Brinda clases base para Modelos, Colecciones y Vistas
  • Permite escuchar y lanzar eventos custom
  • La información se representa con instancias de modelo, cuando la misma cambia, se lanza un evento change
  • Las vistas que muestran el estado del objeto son notificadas para que actualicen la información mostrada
  • Permite conectar nuestros modelos y colecciones con APIs RESTful JSON

Marionette



  • Ayuda a simplificar el desarrollo de grandes aplicaciones hechas con Backbone JS
  • Provee una colección de patrones de diseño
  • Reduce el código boilerplate en las Views
  • Provee Regions, Layouts, CompositeViews y ItemViews

Underscore JS



  • Provee features del estilo de programación funcional
  • Funciones map, select, invoke
  • Provee un motor de template:
    _.template( htmlMarkup, { name: 'Pepe' } ) Ejemplo
  • En browsers modernos (forEach, map, reduce, filter, etc) se delegan a las implementaciones nativas

Namespaces



  • Util para modularizar la creacion de clases
  • No Existe en JS, se la debe simular, colgando la definicion de las clases de window (o global)

Manejo de Dependencias


  • Se utilizan librerias para poder modularizar el codigo en diferentes archivo (tanto JS como HTML)
  • RequireJS: script loader para carga en forma paralela de multiples archivos JS

  • LABjs: otro script loader.
  • CommonJS: Especificación e implementación para crear dependencias con JS

Templates HTML

  • Tags 'script' con referencias a los archivos con template:
    • Se genera una linea de 'script' por cada template.
    • Se sobrecarga el DOM con los templates aunque no esten siendo usados.
    • Se cargan todos los archivos al inicio.

  • Pedido de archivos con requireJS:
    • Se piden los templates a medida que se necesitan.
    • Se genera overhead por cada transicion que haya.

  • Archivo con templates en (forma de String) precompilados:
    • Se puede simular la estructura de directorios en JSON.
    • No se sobrecarga el DOM.
    • Se hace un solo pedido para todos los templates.
